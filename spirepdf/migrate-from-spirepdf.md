## Why Migrate from Spire.PDF

### Critical Technical Issues

Spire.PDF has several fundamental problems that affect real-world usage:

| Issue | Impact | IronPDF Solution |
|-------|--------|------------------|
| **Text rendered as images** | PDFs not searchable, not accessible, can't copy text | Real text rendering |
| **Internet Explorer dependency** | Outdated rendering, security risks | Modern Chromium engine |
| **Font embedding failures** | Documents look wrong on other systems | Reliable font handling |
| **Large deployment footprint** | High memory usage, slow startup | Efficient deployment |
| **Limited CSS support** | Modern layouts don't render correctly | Full CSS3 support |

### The Core Problem: Image-Based PDFs

When you use Spire.PDF's `LoadFromHTML()` method, it often **renders text as bitmap images** rather than actual text. This creates serious problems:

```csharp
// ❌ Spire.PDF - Creates image-based PDF
PdfDocument pdf = new PdfDocument();
pdf.LoadFromHTML("<h1>Important Contract</h1>", false, true, true);
pdf.SaveToFile("contract.pdf");

// Problems with resulting PDF:
// - Text CANNOT be selected
// - Text CANNOT be searched
// - Text CANNOT be copied
// - Screen readers CANNOT read it (accessibility violation)
// - File size is MUCH larger
// - Zooming causes pixelation
```

### Feature Comparison Overview

| Feature | Spire.PDF | IronPDF |
|---------|-----------|---------|
| **HTML Rendering** | IE/Edge-based (outdated) | Chromium (modern) |
| **Text Output** | Images (not selectable) | Real text (selectable) |
| **CSS3 Support** | Limited | Full |
| **Flexbox/Grid** | Not supported | Full support |
| **JavaScript** | Limited | Full ES6+ |
| **Font Embedding** | Problematic | Reliable |
| **PDF Accessibility** | Poor (image-based) | Excellent |
| **Modern .NET** | .NET 6+ partial | Full .NET 6-9 |
| **API Design** | Complex | Simple and intuitive |

## The Text-as-Images Problem

### Why This Is Critical

When Spire.PDF converts HTML to PDF using image-based rendering, your documents lose essential functionality:

**1. No Text Search**
```csharp
// PDF generated by Spire.PDF LoadFromHTML
// Users CANNOT use Ctrl+F to find text
// Document management systems CANNOT index content
// Legal discovery tools CANNOT search documents
```

**2. No Text Selection/Copy**
```csharp
// Users trying to copy a quote, reference, or data:
// 1. Select text → Nothing happens (it's an image)
// 2. Right-click → No "Copy Text" option
// 3. Must manually retype everything
```

**3. Accessibility Violations**
```csharp
// Image-based PDFs fail:
// - WCAG 2.1 compliance
// - Section 508 compliance (US Government)
// - ADA requirements
// - Screen reader compatibility
// Your organization may face legal liability!
```

**4. Large File Sizes**
```csharp
// Same content comparison:
// Spire.PDF (image-based): 2.5 MB
// IronPDF (text-based): 150 KB
// That's 16x larger!
```

### Detection: Is Your PDF Image-Based?

```csharp
// Open your Spire.PDF-generated document
// Try these tests:

// Test 1: Text Selection
// Click and drag over text
// If nothing highlights → IMAGE-BASED

// Test 2: Ctrl+F Search
// Search for any word on the page
// If "No matches found" → IMAGE-BASED

// Test 3: Copy/Paste
// Select and copy text to notepad
// If nothing pastes → IMAGE-BASED
```

### The IronPDF Solution

```csharp
using IronPdf;

// ✅ IronPDF - Creates real text PDF
var renderer = new ChromePdfRenderer();
var pdf = renderer.RenderHtmlAsPdf("<h1>Important Contract</h1>");
pdf.SaveAs("contract.pdf");

// Result:
// ✅ Text is fully selectable
// ✅ Text is searchable with Ctrl+F
// ✅ Text can be copied to clipboard
// ✅ Screen readers work perfectly
// ✅ File size is compact
// ✅ Zooming is crystal clear
```

## The Internet Explorer Problem

### Spire.PDF's Rendering Engine

Spire.PDF relies on **Internet Explorer/Edge Legacy** for HTML rendering in many scenarios:

```csharp
// ❌ Spire.PDF - Uses IE/Edge rendering
// Problems:
// 1. IE was deprecated in 2022
// 2. Modern CSS doesn't work
// 3. JavaScript support is limited
// 4. Security vulnerabilities
// 5. Inconsistent rendering across systems
```

### Modern CSS That Fails in Spire.PDF

```html
<!-- This HTML renders incorrectly in Spire.PDF -->
<div style="display: flex; justify-content: space-between; gap: 20px;">
    <div style="flex: 1;">Column 1</div>
    <div style="flex: 1;">Column 2</div>
</div>

<div style="display: grid; grid-template-columns: repeat(3, 1fr); gap: 10px;">
    <div>Item 1</div>
    <div>Item 2</div>
    <div>Item 3</div>
</div>

<!-- CSS Variables don't work -->
<style>
:root { --primary-color: #007bff; }
h1 { color: var(--primary-color); }
</style>
```

### IronPDF's Modern Chromium Engine

```csharp
using IronPdf;

// ✅ IronPDF - Uses modern Chromium rendering
var renderer = new ChromePdfRenderer();

var html = @"
<style>
    :root { --primary: #007bff; }
    .container { display: flex; gap: 20px; }
    .grid { display: grid; grid-template-columns: repeat(3, 1fr); }
</style>
<div class='container'>
    <div style='flex: 1; color: var(--primary)'>Column 1</div>
    <div style='flex: 1'>Column 2</div>
</div>
<div class='grid'>
    <div>Item 1</div>
    <div>Item 2</div>
    <div>Item 3</div>
</div>";

var pdf = renderer.RenderHtmlAsPdf(html);
// All modern CSS features render correctly!
```

## Quick Start Migration (5 Minutes)

### Step 1: Update NuGet Packages

```bash
# Remove Spire.PDF
dotnet remove package Spire.PDF
dotnet remove package FreeSpire.PDF  # If using free version

# Install IronPDF
dotnet add package IronPdf
```

### Step 2: Update Using Statements

```csharp
// Before
using Spire.Pdf;
using Spire.Pdf.Graphics;
using Spire.Pdf.HtmlConverter;

// After
using IronPdf;
```

### Step 3: Add License Key

```csharp
// Add at application startup
IronPdf.License.LicenseKey = "YOUR-LICENSE-KEY";
```

### Step 4: Update Code

**Before (Spire.PDF):**
```csharp
using Spire.Pdf;
using Spire.Pdf.HtmlConverter;

// HTML to PDF (creates image-based PDF)
PdfDocument pdf = new PdfDocument();
pdf.LoadFromHTML("<h1>Hello World</h1>", false, true, true);
pdf.SaveToFile("output.pdf");
pdf.Close();
```

**After (IronPDF):**
```csharp
using IronPdf;

// HTML to PDF (creates real text PDF)
var renderer = new ChromePdfRenderer();
var pdf = renderer.RenderHtmlAsPdf("<h1>Hello World</h1>");
pdf.SaveAs("output.pdf");
```

## Complete API Reference

### Namespace Mapping

| Spire.PDF Namespace | IronPDF Namespace | Notes |
|---------------------|-------------------|-------|
| `Spire.Pdf` | `IronPdf` | Main namespace |
| `Spire.Pdf.Graphics` | `IronPdf.Rendering` | Rendering options |
| `Spire.Pdf.HtmlConverter` | `IronPdf` | `ChromePdfRenderer` |
| `Spire.Pdf.Widget` | `IronPdf.Forms` | Form handling |
| `Spire.Pdf.Security` | `IronPdf` | Security settings |
| `Spire.Pdf.Annotations` | `IronPdf` | PDF annotations |

### Core Class Mapping

| Spire.PDF | IronPDF | Notes |
|-----------|---------|-------|
| `PdfDocument` | `PdfDocument` | Same name, different API |
| `PdfDocument.LoadFromHTML()` | `ChromePdfRenderer.RenderHtmlAsPdf()` | Real text output |
| `PdfDocument.LoadFromFile()` | `PdfDocument.FromFile()` | Static method |
| `PdfDocument.SaveToFile()` | `PdfDocument.SaveAs()` | Fluent API |
| `PdfDocument.Close()` | Not needed | Dispose pattern |
| `PdfHTMLLayoutFormat` | `ChromePdfRenderOptions` | Rendering options |
| `PdfPageBase` | `PdfDocument.Pages[n]` | Page access |
| `PdfFont`/`PdfTrueTypeFont` | CSS fonts | HTML/CSS styling |
| `PdfBrush`/`PdfPen` | CSS styling | HTML/CSS approach |

### HTML Conversion Options Mapping

| Spire.PDF Option | IronPDF Equivalent |
|------------------|-------------------|
| `LoadFromHTML(html, ...)` | `RenderHtmlAsPdf(html)` |
| `PdfHTMLLayoutFormat.Layout = FitToPage` | `FitToPaperMode = AutomaticZoom` |
| `PdfHTMLLayoutFormat.IsWaiting = true` | `WaitFor.JavaScript(ms)` |
| Browser width setting | `PaperSize` + `ViewPortWidth` |
| Custom page size | `SetCustomPaperSizeInInches()` |

### Page Settings Mapping

| Spire.PDF | IronPDF |
|-----------|---------|
| `PdfPageSettings.Width` | `PaperSize` or custom size |
| `PdfPageSettings.Height` | `PaperSize` or custom size |
| `PdfPageSettings.Margins` | Individual `MarginTop/Bottom/Left/Right` |
| `PdfPageSettings.Orientation.Landscape` | `PaperOrientation = Landscape` |
| `PdfPageSettings.Orientation.Portrait` | `PaperOrientation = Portrait` |

### Security Settings Mapping

| Spire.PDF Security | IronPDF Equivalent |
|--------------------|-------------------|
| `PdfSecurity.OwnerPassword` | `SecuritySettings.OwnerPassword` |
| `PdfSecurity.UserPassword` | `SecuritySettings.UserPassword` |
| `PdfPermissionsFlags.Print` | `AllowUserPrinting` |
| `PdfPermissionsFlags.CopyContent` | `AllowUserCopyPasteContent` |
| `PdfPermissionsFlags.Edit` | `AllowUserEdits` |
| `PdfPermissionsFlags.FillFields` | `AllowUserFormData` |

### Example 4: Headers and Footers

**Before (Spire.PDF):**
```csharp
using Spire.Pdf;
using Spire.Pdf.Graphics;
using System.Drawing;

class Program
{
    static void Main()
    {
        PdfDocument pdf = new PdfDocument();
        pdf.LoadFromHTML("<h1>Content</h1>", false, true, true);

        // Manual header/footer for each page
        foreach (PdfPageBase page in pdf.Pages)
        {
            PdfFont font = new PdfFont(PdfFontFamily.Helvetica, 10);
            PdfBrush brush = PdfBrushes.Black;

            // Header
            page.Canvas.DrawString("My Document", font, brush,
                new PointF(page.Canvas.ClientSize.Width / 2, 10));

            // Footer
            page.Canvas.DrawString($"Page {pdf.Pages.IndexOf(page) + 1}", font, brush,
                new PointF(page.Canvas.ClientSize.Width / 2, page.Canvas.ClientSize.Height - 20));
        }

        pdf.SaveToFile("output.pdf");
        pdf.Close();
    }
}
```

**After (IronPDF):**
```csharp
using IronPdf;

class Program
{
    static void Main()
    {
        var renderer = new ChromePdfRenderer();

        // Headers and footers with full HTML support
        renderer.RenderingOptions.HtmlHeader = new HtmlHeaderFooter()
        {
            HtmlFragment = @"
                <div style='width:100%; text-align:center; font-family:Arial; font-size:10px;
                            border-bottom:1px solid #ccc; padding-bottom:5px;'>
                    My Document
                </div>"
        };

        renderer.RenderingOptions.HtmlFooter = new HtmlHeaderFooter()
        {
            HtmlFragment = @"
                <div style='width:100%; text-align:center; font-family:Arial; font-size:10px;'>
                    Page {page} of {total-pages}
                </div>"
        };

        // Ensure space for headers/footers
        renderer.RenderingOptions.MarginTop = 25;
        renderer.RenderingOptions.MarginBottom = 20;

        var pdf = renderer.RenderHtmlAsPdf("<h1>Content</h1>");
        pdf.SaveAs("output.pdf");
    }
}
```

### Example 5: Merge PDFs

**Before (Spire.PDF):**
```csharp
using Spire.Pdf;

class Program
{
    static void Main()
    {
        PdfDocument pdf1 = new PdfDocument();
        pdf1.LoadFromFile("document1.pdf");

        PdfDocument pdf2 = new PdfDocument();
        pdf2.LoadFromFile("document2.pdf");

        // Insert pages from pdf2 into pdf1
        pdf1.InsertPageRange(pdf2, 0, pdf2.Pages.Count - 1);

        pdf1.SaveToFile("merged.pdf");
        pdf1.Close();
        pdf2.Close();
    }
}
```

**After (IronPDF):**
```csharp
using IronPdf;

class Program
{
    static void Main()
    {
        var pdf1 = PdfDocument.FromFile("document1.pdf");
        var pdf2 = PdfDocument.FromFile("document2.pdf");

        // Clean merge with single method
        var merged = PdfDocument.Merge(pdf1, pdf2);

        merged.SaveAs("merged.pdf");
    }
}
```

### Example 7: Add Text Stamp/Annotation

**Before (Spire.PDF):**
```csharp
using Spire.Pdf;
using Spire.Pdf.Graphics;
using System.Drawing;

class Program
{
    static void Main()
    {
        PdfDocument pdf = new PdfDocument();
        pdf.LoadFromFile("document.pdf");

        foreach (PdfPageBase page in pdf.Pages)
        {
            PdfFont font = new PdfFont(PdfFontFamily.Helvetica, 48);
            PdfBrush brush = new PdfSolidBrush(Color.FromArgb(50, 255, 0, 0));

            // Rotate and draw watermark
            PdfGraphicsState state = page.Canvas.Save();
            page.Canvas.RotateTransform(-45);
            page.Canvas.DrawString("CONFIDENTIAL", font, brush,
                new PointF(0, page.Canvas.ClientSize.Height / 2));
            page.Canvas.Restore(state);
        }

        pdf.SaveToFile("watermarked.pdf");
        pdf.Close();
    }
}
```

**After (IronPDF):**

## Troubleshooting Guide

### Issue 1: Text Not Selectable in Old PDFs

**Problem:** PDFs created with Spire.PDF's `LoadFromHTML` have image-based text

**Solution:** Regenerate with IronPDF:
```csharp
// Cannot fix existing image-based PDFs
// Must regenerate from source HTML

var renderer = new ChromePdfRenderer();
var pdf = renderer.RenderHtmlAsPdf(originalHtml);
pdf.SaveAs("fixed.pdf");
// Now text is selectable!
```

### Issue 2: Different Layout After Migration

**Problem:** Layouts look different due to rendering engine change

**Solution:** IronPDF uses modern Chromium, which renders CSS correctly:
```csharp
var renderer = new ChromePdfRenderer();

// If layout needs adjustment:
renderer.RenderingOptions.ViewPortWidth = 1280; // Match desired width
renderer.RenderingOptions.CssMediaType = PdfCssMediaType.Print; // Or Screen
renderer.RenderingOptions.Zoom = 100; // Adjust scaling

var pdf = renderer.RenderHtmlAsPdf(html);
```

### Issue 3: Missing Fonts

**Problem:** Fonts look different or missing

**Solution:** IronPDF handles fonts automatically, but you can embed:
```csharp
// Include font in HTML
var html = @"
<style>
@import url('https://fonts.googleapis.com/css2?family=Roboto:wght@400;700&display=swap');
body { font-family: 'Roboto', sans-serif; }
</style>
<body>Content with custom font</body>";

var renderer = new ChromePdfRenderer();
var pdf = renderer.RenderHtmlAsPdf(html);
```

### Issue 4: JavaScript Not Executing

**Problem:** Dynamic content not appearing

**Solution:** Enable JavaScript and wait:
```csharp
var renderer = new ChromePdfRenderer();
renderer.RenderingOptions.EnableJavaScript = true;
renderer.RenderingOptions.WaitFor.JavaScript(2000);

// Or wait for specific element
renderer.RenderingOptions.WaitFor.HtmlElementById("content-loaded", 5000);

var pdf = renderer.RenderHtmlAsPdf(htmlWithJs);
```

### Issue 5: API Pattern Changes

**Problem:** Code structure is different

**Solution:** Key pattern changes:
```csharp
// Spire.PDF Pattern
PdfDocument pdf = new PdfDocument();
pdf.LoadFromHTML(...);
pdf.SaveToFile(...);
pdf.Close();

// IronPDF Pattern
var renderer = new ChromePdfRenderer();
var pdf = renderer.RenderHtmlAsPdf(...);
pdf.SaveAs(...);
// No Close() needed - uses dispose pattern
```

## Migration Checklist

### Pre-Migration

- [ ] **Inventory all Spire.PDF usages**
  ```bash
  # Find all Spire.PDF usages
  grep -r "Spire\.Pdf\|LoadFromHTML\|LoadFromFile\|SaveToFile" --include="*.cs" .

  # Count critical LoadFromHTML calls (these are the priority)
  grep -r "LoadFromHTML" --include="*.cs" . | wc -l
  ```
  **Why:** Understand scope and identify the most critical issue: `LoadFromHTML()` converts text to images.

- [ ] **Test existing PDFs for text selectability**
  ```csharp
  // CRITICAL: Test if text is selectable in your existing PDFs
  // Open a Spire-generated PDF and try to select/copy text
  // If you can't select text, you're hitting Spire's major limitation

  // After migration, all text will be selectable!
  var renderer = new ChromePdfRenderer();
  var pdf = renderer.RenderHtmlAsPdf("<p>Test selectable text</p>");
  ```
  **Why:** Spire.PDF's `LoadFromHTML()` creates image-based PDFs—text is NOT selectable.

### Package Changes

- [ ] **Remove Spire.PDF package**
  ```bash
  dotnet remove package Spire.PDF
  dotnet remove package FreeSpire.PDF  # If using free version
  ```
  **Why:** Eliminate the image-based rendering problem.

- [ ] **Install IronPDF**
  ```bash
  dotnet add package IronPdf
  ```
  **Why:** True HTML rendering with selectable, searchable text.

### Code Updates

- [ ] **Update namespace imports**
  ```csharp
  // Before
  using Spire.Pdf;
  using Spire.Pdf.Graphics;
  using Spire.Pdf.Widget;

  // After
  using IronPdf;
  ```
  **Why:** Clean namespace transition.

- [ ] **Add license initialization**
  ```csharp
  IronPdf.License.LicenseKey = Environment.GetEnvironmentVariable("IRONPDF_LICENSE_KEY");
  ```
  **Why:** Configure before first use.

- [ ] **Replace LoadFromHTML with RenderHtmlAsPdf (CRITICAL FIX)**
  ```csharp
  // Before (Spire.PDF) - Creates IMAGE-BASED PDFs (text not selectable!)
  var doc = new PdfDocument();
  var settings = new PdfHtmlLayoutFormat();
  settings.IsWaiting = true;
  PdfDocument.LoadFromHTML(htmlContent, true, true, true);
  doc.SaveToFile("output.pdf");
  doc.Close();

  // After (IronPDF) - Creates TRUE PDF with selectable text
  var renderer = new ChromePdfRenderer();
  var pdf = renderer.RenderHtmlAsPdf(htmlContent);
  pdf.SaveAs("output.pdf");
  // Text is now selectable, searchable, and accessible!
  ```
  **Why:** This is the most important change—fixes the image-based text problem.

- [ ] **Replace PdfDocument creation pattern**
  ```csharp
  // Before (Spire.PDF)

  doc.LoadFromFile("input.pdf");
  // ... operations
  doc.SaveToFile("output.pdf");
  doc.Close();

  // After (IronPDF)
  var pdf = PdfDocument.FromFile("input.pdf");
  // ... operations
  pdf.SaveAs("output.pdf");
  // No Close() needed - use 'using' if desired
  ```
  **Why:** Simpler object model.

- [ ] **Update margin configurations**
  ```csharp

  // After (IronPDF)
  renderer.RenderingOptions.MarginTop = 18;    // mm (approx 50 points)
  renderer.RenderingOptions.MarginBottom = 18;
  renderer.RenderingOptions.MarginLeft = 18;
  renderer.RenderingOptions.MarginRight = 18;
  ```
  **Why:** IronPDF uses millimeters for intuitive measurement.

- [ ] **Update header/footer implementations**
  ```csharp
  // Before (Spire.PDF)
  var section = doc.Sections.Add();
  var header = section.PageSettings.Header;
  header.SetValue("Header text");
  var footer = section.PageSettings.Footer;
  footer.SetValue("Page {pageNumber}");

  // After (IronPDF)
  renderer.RenderingOptions.HtmlHeader = new HtmlHeaderFooter
  {
      HtmlFragment = "<div style='text-align:center;'>Header text</div>",
      MaxHeight = 30
  };
  renderer.RenderingOptions.HtmlFooter = new HtmlHeaderFooter
  {
      HtmlFragment = "<div style='text-align:center;'>Page {page} of {total-pages}</div>",
      MaxHeight = 25
  };
  ```
  **Why:** HTML-based headers are more flexible.

- [ ] **Update watermark implementations**
  ```csharp
  // Before (Spire.PDF)
  var font = new PdfFont(PdfFontFamily.Helvetica, 48);
  var brush = new PdfSolidBrush(new PdfRGBColor(255, 0, 0));
  foreach (PdfPageBase page in doc.Pages)
  {
      page.Canvas.SetTransparency(0.5f);
      page.Canvas.RotateTransform(-45);
      page.Canvas.DrawString("DRAFT", font, brush, 100, 400);
  }

  // After (IronPDF)
  var pdf = renderer.RenderHtmlAsPdf(html);
  pdf.ApplyWatermark(@"
      <div style='
          font-size: 48px;
          color: rgba(255, 0, 0, 0.5);
          transform: rotate(-45deg);
      '>DRAFT</div>");
  ```
  **Why:** CSS-based watermarks are more powerful.
