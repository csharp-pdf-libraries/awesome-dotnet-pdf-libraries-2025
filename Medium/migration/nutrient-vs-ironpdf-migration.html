I've helped dozens of .NET teams migrate their PDF stacks. The Nutrient (formerly PSPDFKit) ones always share the same complaint: "we're paying platform prices for library-sized needs."

Nutrient started life as PSPDFKit — a mobile-first PDF viewer SDK. By 2024 it had rebranded and repositioned as a "document intelligence platform," which is a reasonable direction for enterprise buyers but frustrating if you just need reliable HTML-to-PDF, merging, and watermarking in a .NET backend. The async-everywhere API and opaque pricing round out a picture that makes switching look attractive.

This guide covers the concrete code changes. Every Before block reflects the PSPDFKit.Pdf SDK that Nutrient .NET developers have been shipping with. Every After block is IronPDF.

Why Teams Switch

The practical friction points come up in almost every migration conversation:

  • Platform overengineering. AI features and document workflow layers you did not ask for raise complexity and binary footprint without adding value for pure PDF work.
  • Async-first tax. PdfProcessor.CreateAsync() is not optional — every operation flows through it. That overhead is fine in an event-driven architecture, but it is boilerplate in a synchronous service.
  • Pricing opacity. Nutrient quotes on request. IronPDF publishes dollar figures. Teams that need to justify budget a week before a sprint cannot wait for a sales call.
  • Rebrand confusion. Package names still carry PSPDFKit; docs sometimes say Nutrient. New hires searching for answers land in the wrong era of documentation.

The full migration guide on ironpdf.com covers licensing detail and edge cases beyond what fits here.

Namespace Changes

  • using PSPDFKit.Pdf; → using IronPdf; — core namespace
  • using PSPDFKit.Pdf.Document; → using IronPdf;
  • using PSPDFKit.Pdf.Rendering; → using IronPdf.Rendering;
  • using PSPDFKit.Pdf.Forms; → using IronPdf.Forms;
  • using PSPDFKit.Pdf.Annotation; → using IronPdf; — watermarks become HTML
  • using Nutrient.Pdf; → using IronPdf;

Complete API Mapping

Initialization

  • await PdfProcessor.CreateAsync() → new ChromePdfRenderer() — no async required
  • processor.Dispose() → automatic or manual — simpler lifecycle
  • new PdfConfiguration { ... } → renderer.RenderingOptions — property-based config

Document Loading

  • await processor.OpenAsync(path) → PdfDocument.FromFile(path) — sync by default
  • Document.LoadFromStream(stream) → PdfDocument.FromStream(stream)
  • Document.LoadFromBytes(bytes) → new PdfDocument(bytes)

PDF Generation

  • await processor.GeneratePdfFromHtmlStringAsync(html) → renderer.RenderHtmlAsPdf(html) — sync
  • await processor.GeneratePdfFromUrlAsync(url) → renderer.RenderUrlAsPdf(url)
  • await processor.GeneratePdfFromFileAsync(path) → renderer.RenderHtmlFileAsPdf(path)

Page Configuration

  • config.PageSize = PageSize.A4 → RenderingOptions.PaperSize = PdfPaperSize.A4
  • config.Orientation = Orientation.Landscape → RenderingOptions.PaperOrientation = PdfPaperOrientation.Landscape
  • config.Margins = new Margins(t, r, b, l) → individual MarginTop, MarginRight, MarginBottom, MarginLeft properties

Document Operations

  • await processor.MergeAsync(docs) → PdfDocument.Merge(pdfs) — sync, static method
  • document.PageCount → pdf.PageCount — same pattern
  • document.GetPages() → pdf.Pages — collection
  • document.AddPage(page) → pdf.AppendPdf(otherPdf)
  • await document.RemovePageAsync(index) → pdf.RemovePages(index) — sync

Annotations and Watermarks

  • await document.AddAnnotationAsync(index, annotation) → pdf.ApplyWatermark(html, rotation, vAlign, hAlign) — HTML-based, single call covers all pages
  • new TextAnnotation("text") → HTML string in watermark — more flexible
  • annotation.Opacity = 0.5 → CSS opacity: 0.5 inline on the HTML element
  • annotation.FontSize = 48 → CSS font-size: 48px

Form Handling

  • document.GetFormFields() → pdf.Form.Fields
  • field.SetValue(value) → field.Value = value — property setter
  • document.FlattenAnnotations() → pdf.Form.Flatten()

Headers and Footers

  • Nutrient: complex annotation loop per page → RenderingOptions.HtmlHeader set once
  • Manual $"Page {i + 1}" string per page → {page} and {total-pages} placeholders built in

Output

  • await document.SaveAsync(path) → pdf.SaveAs(path) — sync
  • document.ToBytes() → pdf.BinaryData
  • document.ToStream() → pdf.Stream

Code Migration Examples

Example 1: HTML to PDF

// Before — Nutrient / PSPDFKit
// NuGet: Install-Package PSPDFKit.Dotnet
using PSPDFKit.Pdf;
using System.Threading.Tasks;

class Program
{
    static async Task Main()
    {
        var htmlContent = "<html><body><h1>Hello World</h1></body></html>";

        using var processor = await PdfProcessor.CreateAsync();
        var document = await processor.GeneratePdfFromHtmlStringAsync(htmlContent);
        await document.SaveAsync("output.pdf");
    }
}

// After — IronPDF
// NuGet: Install-Package IronPdf
using IronPdf;

class Program
{
    static void Main()
    {
        var htmlContent = "<html><body><h1>Hello World</h1></body></html>";

        var renderer = new ChromePdfRenderer();
        var pdf = renderer.RenderHtmlAsPdf(htmlContent);
        pdf.SaveAs("output.pdf");
    }
}

The async ceremony disappears. ChromePdfRenderer is synchronous by default and reusable — instantiate it once, call it many times.

Example 2: URL to PDF with Configuration

// Before — Nutrient / PSPDFKit
using PSPDFKit.Pdf;
using System.Threading.Tasks;

public async Task<byte[]> ConvertUrlAsync(string url)
{
    var configuration = new PdfConfiguration
    {
        PageSize = PageSize.A4,
        Orientation = Orientation.Portrait,
        Margins = new Margins(20, 20, 20, 20),
        JavaScriptEnabled = true,
        WaitForLoadingComplete = true
    };

    using var processor = await PdfProcessor.CreateAsync();
    var document = await processor.CreatePdfFromUrlAsync(url, configuration);
    return await document.ToBytesAsync();
}

// After — IronPDF
using IronPdf;

public byte[] ConvertUrl(string url)
{
    var renderer = new ChromePdfRenderer();

    renderer.RenderingOptions.PaperSize = PdfPaperSize.A4;
    renderer.RenderingOptions.PaperOrientation = PdfPaperOrientation.Portrait;
    renderer.RenderingOptions.MarginTop = 20;
    renderer.RenderingOptions.MarginBottom = 20;
    renderer.RenderingOptions.MarginLeft = 20;
    renderer.RenderingOptions.MarginRight = 20;
    renderer.RenderingOptions.EnableJavaScript = true;
    renderer.RenderingOptions.WaitFor.RenderDelay(2000);

    var pdf = renderer.RenderUrlAsPdf(url);
    return pdf.BinaryData;
}

Example 3: Merging PDFs

// Before — Nutrient / PSPDFKit
// NuGet: Install-Package PSPDFKit.Dotnet
using PSPDFKit.Pdf;
using System.Collections.Generic;
using System.Threading.Tasks;

class Program
{
    static async Task Main()
    {
        using var processor = await PdfProcessor.CreateAsync();

        var document1 = await processor.OpenAsync("document1.pdf");
        var document2 = await processor.OpenAsync("document2.pdf");

        var mergedDocument = await processor.MergeAsync(
            new List<PdfDocument> { document1, document2 });
        await mergedDocument.SaveAsync("merged.pdf");
    }
}

// After — IronPDF
// NuGet: Install-Package IronPdf
using IronPdf;
using System.Collections.Generic;

class Program
{
    static void Main()
    {
        var pdf1 = PdfDocument.FromFile("document1.pdf");
        var pdf2 = PdfDocument.FromFile("document2.pdf");

        var merged = PdfDocument.Merge(pdf1, pdf2);
        merged.SaveAs("merged.pdf");
    }
}

The processor lifecycle goes away entirely. PdfDocument.Merge is a static method — pass two documents or a collection, get one back.

Example 4: Adding Watermarks

// Before — Nutrient / PSPDFKit
// NuGet: Install-Package PSPDFKit.Dotnet
using PSPDFKit.Pdf;
using PSPDFKit.Pdf.Annotation;
using System.Threading.Tasks;

class Program
{
    static async Task Main()
    {
        using var processor = await PdfProcessor.CreateAsync();
        var document = await processor.OpenAsync("document.pdf");

        for (int i = 0; i < document.PageCount; i++)
        {
            var watermark = new TextAnnotation("CONFIDENTIAL")
            {
                Opacity = 0.5,
                FontSize = 48
            };
            await document.AddAnnotationAsync(i, watermark);
        }

        await document.SaveAsync("watermarked.pdf");
    }
}

// After — IronPDF
// NuGet: Install-Package IronPdf
using IronPdf;
using IronPdf.Editing;

class Program
{
    static void Main()
    {
        var pdf = PdfDocument.FromFile("document.pdf");

        pdf.ApplyWatermark(
            "<h1 style='color:gray;opacity:0.5;'>CONFIDENTIAL</h1>",
            50,
            VerticalAlignment.Middle,
            HorizontalAlignment.Center);

        pdf.SaveAs("watermarked.pdf");
    }
}

The per-page annotation loop collapses to a single call. The HTML string gives you full CSS control over font, color, and opacity. The rotation argument is an integer (degrees).

Example 5: Headers and Footers

// Before — Nutrient / PSPDFKit
// Nutrient requires adding text annotations to each page individually
using PSPDFKit.Pdf;
using System.Threading.Tasks;

public async Task CreateWithHeaderFooterAsync(string html, string outputPath)
{
    using var processor = await PdfProcessor.CreateAsync();
    var document = await processor.GeneratePdfFromHtmlStringAsync(html);

    for (int i = 0; i < document.PageCount; i++)
    {
        var header = new TextAnnotation("Document Header")
        {
            Position = new Position(document.GetPage(i).Width / 2, 20),
            FontSize = 10
        };
        await document.AddAnnotationAsync(i, header);

        var footer = new TextAnnotation($"Page {i + 1}")
        {
            Position = new Position(
                document.GetPage(i).Width / 2,
                document.GetPage(i).Height - 20),
            FontSize = 10
        };
        await document.AddAnnotationAsync(i, footer);
    }

    await document.SaveAsync(outputPath);
}

// After — IronPDF
using IronPdf;

public void CreateWithHeaderFooter(string html, string outputPath)
{
    var renderer = new ChromePdfRenderer();

    renderer.RenderingOptions.HtmlHeader = new HtmlHeaderFooter
    {
        HtmlFragment = "<div style='text-align:center; font-size:10px;'>Document Header</div>",
        MaxHeight = 25
    };

    renderer.RenderingOptions.HtmlFooter = new HtmlHeaderFooter
    {
        HtmlFragment = "<div style='text-align:center; font-size:10px;'>Page {page} of {total-pages}</div>",
        MaxHeight = 25
    };

    var pdf = renderer.RenderHtmlAsPdf(html);
    pdf.SaveAs(outputPath);
}

The {page} and {total-pages} placeholders are resolved at render time. No per-page loop required.

Example 6: Form Field Handling

// Before — Nutrient / PSPDFKit
using PSPDFKit.Pdf;
using PSPDFKit.Pdf.Forms;
using System.Threading.Tasks;

public async Task FillFormAsync(string inputPath, string outputPath)
{
    using var processor = await PdfProcessor.CreateAsync();
    var document = await processor.OpenAsync(inputPath);

    var formFields = await document.GetFormFieldsAsync();

    foreach (var field in formFields)
    {
        if (field.Name == "CustomerName")
            await field.SetValueAsync("John Doe");
        else if (field.Name == "Date")
            await field.SetValueAsync(DateTime.Now.ToString("yyyy-MM-dd"));
    }

    await document.SaveAsync(outputPath);
}

// After — IronPDF
using IronPdf;

public void FillForm(string inputPath, string outputPath)
{
    var pdf = PdfDocument.FromFile(inputPath);

    pdf.Form.FindFormField("CustomerName").Value = "John Doe";
    pdf.Form.FindFormField("Date").Value = DateTime.Now.ToString("yyyy-MM-dd");

    pdf.SaveAs(outputPath);
}

Example 7: Password Protection

// Before — Nutrient / PSPDFKit
using PSPDFKit.Pdf;
using PSPDFKit.Pdf.Security;
using System.Threading.Tasks;

public async Task ProtectPdfAsync(string inputPath, string outputPath, string password)
{
    using var processor = await PdfProcessor.CreateAsync();
    var document = await processor.OpenAsync(inputPath);

    var securityOptions = new SecurityOptions
    {
        OwnerPassword = password,
        UserPassword = password,
        Permissions = new Permissions
        {
            Printing = PrintingPermission.HighResolution,
            ModifyContent = false,
            CopyContent = false
        }
    };

    await document.ApplySecurityAsync(securityOptions);
    await document.SaveAsync(outputPath);
}

// After — IronPDF
using IronPdf;

public void ProtectPdf(string inputPath, string outputPath, string password)
{
    var pdf = PdfDocument.FromFile(inputPath);

    pdf.SecuritySettings.OwnerPassword = password;
    pdf.SecuritySettings.UserPassword = password;
    pdf.SecuritySettings.AllowUserPrinting = PdfPrintSecurity.FullPrintRights;
    pdf.SecuritySettings.AllowUserEdits = PdfEditSecurity.NoEdit;
    pdf.SecuritySettings.AllowUserCopyPasteContent = false;

    pdf.SaveAs(outputPath);
}

Example 8: Text Extraction

// Before — Nutrient / PSPDFKit
using PSPDFKit.Pdf;
using System.Text;
using System.Threading.Tasks;

public async Task<string> ExtractTextAsync(string pdfPath)
{
    using var processor = await PdfProcessor.CreateAsync();
    var document = await processor.OpenAsync(pdfPath);

    var sb = new StringBuilder();
    for (int i = 0; i < document.PageCount; i++)
    {
        var page = document.GetPage(i);
        var text = await page.ExtractTextAsync();
        sb.AppendLine(text);
    }

    return sb.ToString();
}

// After — IronPDF
using IronPdf;

public string ExtractText(string pdfPath)
{
    var pdf = PdfDocument.FromFile(pdfPath);
    return pdf.ExtractAllText();
}

Example 9: Page Manipulation

// Before — Nutrient / PSPDFKit
using PSPDFKit.Pdf;
using System.Threading.Tasks;

public async Task ManipulatePagesAsync(string inputPath, string outputPath)
{
    using var processor = await PdfProcessor.CreateAsync();
    var document = await processor.OpenAsync(inputPath);

    await document.RemovePageAsync(0);

    var page = document.GetPage(0);
    await page.SetRotationAsync(90);

    await document.SaveAsync(outputPath);
}

// After — IronPDF
using IronPdf;

public void ManipulatePages(string inputPath, string outputPath)
{
    var pdf = PdfDocument.FromFile(inputPath);

    pdf.RemovePages(0);

    pdf.Pages[0].Rotation = PdfPageRotation.Rotate90;

    var reordered = pdf.CopyPages(2, 0, 1);
    reordered.SaveAs(outputPath);
}

Example 10: Complete Workflow

// Before — Nutrient / PSPDFKit
using PSPDFKit.Pdf;
using System.Threading.Tasks;

public class NutrientWorkflow
{

    {
        using var processor = await PdfProcessor.CreateAsync();

        var config = new PdfConfiguration
        {
            PageSize = PageSize.A4,
            Margins = new Margins(20, 20, 20, 20)
        };

        var document = await processor.GeneratePdfFromHtmlStringAsync(html, config);

        for (int i = 0; i < document.PageCount; i++)
        {
            var watermark = new TextAnnotation("DRAFT") { Opacity = 0.3f };
            await document.AddAnnotationAsync(i, watermark);
        }

        await document.ApplySecurityAsync(new SecurityOptions
        {
            OwnerPassword = "admin",
            Permissions = new Permissions { ModifyContent = false }
        });

        await document.SaveAsync(outputPath);
    }
}

// After — IronPDF

using IronPdf.Editing;

public class PdfWorkflow
{

    public PdfWorkflow()
    {

        _renderer.RenderingOptions.PaperSize = PdfPaperSize.A4;
        _renderer.RenderingOptions.MarginTop = 20;
        _renderer.RenderingOptions.MarginBottom = 20;
        _renderer.RenderingOptions.MarginLeft = 20;
        _renderer.RenderingOptions.MarginRight = 20;

        _renderer.RenderingOptions.HtmlFooter = new HtmlHeaderFooter
        {

    public void ProcessDocument(string html, string outputPath)

            VerticalAlignment.Middle,
            HorizontalAlignment.Center);

        pdf.SecuritySettings.OwnerPassword = "admin";
        pdf.SecuritySettings.AllowUserEdits = PdfEditSecurity.NoEdit;

        pdf.SaveAs(outputPath);
    }
}

Common Migration Issues

Issue 1: Async to Sync Pattern

Nutrient's async-first design means every operation — even simple file loads — goes through await. IronPDF is sync by default; async variants are available when you need them.

// Nutrient (async required):
var document = await processor.OpenAsync(path);

// IronPDF (sync by default):
var pdf = PdfDocument.FromFile(path);

// IronPDF (async if needed):
var pdf = await renderer.RenderHtmlAsPdfAsync(html);

Issue 2: Configuration Pattern

Nutrient passes a configuration object into each call. IronPDF uses a RenderingOptions property bag on the renderer, set once and reused.

// Nutrient:
var config = new PdfConfiguration
{
    PageSize = PageSize.A4,
    Margins = new Margins(20, 20, 20, 20)
};
var doc = await processor.GeneratePdfFromHtmlStringAsync(html, config);

// IronPDF:
var renderer = new ChromePdfRenderer();
renderer.RenderingOptions.PaperSize = PdfPaperSize.A4;
renderer.RenderingOptions.MarginTop = 20;
renderer.RenderingOptions.MarginBottom = 20;
var pdf = renderer.RenderHtmlAsPdf(html);

Issue 3: Processor Lifecycle

Nutrient requires creating and disposing a processor around every session. IronPDF's renderer has no mandatory lifecycle — instantiate it, reuse it.

// Nutrient:
using var processor = await PdfProcessor.CreateAsync();
// ... use processor, then dispose

// IronPDF:
var renderer = new ChromePdfRenderer();
// Reuse renderer; no complex lifecycle

Issue 4: Annotation vs HTML Watermarks

Nutrient watermarks are annotation objects with typed properties. IronPDF watermarks are HTML strings styled with CSS — more flexible, no new types to learn.

// Nutrient annotation:
new TextAnnotation("CONFIDENTIAL") { Opacity = 0.5, FontSize = 48 }

// IronPDF HTML:
"<h1 style='opacity:0.5; font-size:48px;'>CONFIDENTIAL</h1>"

Issue 5: Page Numbers in Headers and Footers

Nutrient requires a manual loop that calculates and adds a text annotation to every page. IronPDF resolves {page} and {total-pages} at render time automatically.

// Nutrient: must calculate per-page
for (int i = 0; i < doc.PageCount; i++)
{
    var footer = new TextAnnotation($"Page {i + 1} of {doc.PageCount}");
    await doc.AddAnnotationAsync(i, footer);
}

// IronPDF: built-in placeholders
renderer.RenderingOptions.HtmlFooter = new HtmlHeaderFooter
{
    HtmlFragment = "Page {page} of {total-pages}"
};

Performance Comparison

  • HTML to PDF (simple). Nutrient ~800 ms; IronPDF ~400 ms.
  • HTML to PDF (complex). Nutrient ~2000 ms; IronPDF ~800 ms.
  • Load PDF (10 pages). Nutrient ~150 ms; IronPDF ~100 ms.
  • Merge 2 PDFs. Nutrient ~200 ms; IronPDF ~50 ms.
  • Add watermark. Nutrient ~300 ms; IronPDF ~30 ms.
  • Extract text. Nutrient ~100 ms; IronPDF ~50 ms.
  • First operation (cold init). Nutrient ~2000 ms; IronPDF ~1500 ms.

Migration Checklist

Pre-Migration

  • Inventory all PSPDFKit/Nutrient usages. Run grep -r "PSPDFKit\|Nutrient\|PdfProcessor" --include="*.cs" . to find every file that needs attention before you start.
  • Document async patterns that need adjustment. Every await PdfProcessor.CreateAsync() call is a migration point. List them now.
  • List all configuration objects. PdfConfiguration properties map to RenderingOptions; document each one to ensure consistent output after the switch.
  • Identify annotation-based features. Watermarks and headers built with TextAnnotation need to be rewritten as HTML strings.
  • Review form handling requirements. Confirm which form fields are read vs written and whether flatten is needed.

Migration Steps

  • Install IronPDF NuGet package. dotnet add package IronPdf
  • Remove PSPDFKit/Nutrient packages. dotnet remove package PSPDFKit and dotnet remove package Nutrient
  • Add IronPDF license key at startup. IronPdf.License.LicenseKey = "YOUR-LICENSE-KEY"; — free trial available at ironpdf.com.
  • Update namespace imports. Replace using PSPDFKit.Pdf; with using IronPdf; across all files.
  • Replace PdfProcessor.CreateAsync() with new ChromePdfRenderer(). The renderer is synchronous and reusable.
  • Convert async methods to sync (or keep async variants). IronPDF provides both; simplify where async adds no value.
  • Replace configuration objects with RenderingOptions properties. Set once on the renderer instance.
  • Convert annotation watermarks to HTML watermarks. pdf.ApplyWatermark("<h1 style='color:red; opacity:0.3;'>Confidential</h1>")
  • Update header/footer to use HtmlHeaderFooter with placeholders. Replace per-page annotation loops with a single RenderingOptions.HtmlHeader / HtmlFooter assignment.

Post-Migration

  • Remove async/await where no longer needed. Simplify call sites that were async only because Nutrient required it.
  • Run regression tests comparing PDF output. Verify rendering, layout, and fonts match expectations.
  • Verify headers/footers with page numbers. Confirm {page} and {total-pages} resolve correctly in your documents.
  • Test form filling functionality. Check read, write, and flatten paths.
  • Confirm security settings. Validate password protection and permission flags on generated PDFs.
  • Update CI/CD pipeline. Reflect the new package dependency and remove any PSPDFKit-specific build steps.

Pricing and Licensing

Nutrient positions itself as an enterprise document platform and prices accordingly — figures require a sales conversation. IronPDF publishes its pricing: a single-developer license is available at a flat rate, with team and OEM tiers listed on the ironpdf.com pricing page. Both products offer trial periods; IronPDF's trial gives full functionality with a watermark on output rather than a feature-limited evaluation mode.

For small-to-medium teams that do not need the AI and workflow features Nutrient bundles, the cost difference is significant. For enterprise buyers evaluating the full document intelligence platform, Nutrient may justify its price — but that is a different product decision than choosing a PDF library for backend PDF generation.
