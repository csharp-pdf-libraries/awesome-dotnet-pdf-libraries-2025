From ActivePDF Toolkit to IronPDF: A Practical Migration Guide

I've helped a number of teams move off ActivePDF Toolkit over the past few years — usually prompted by ActivePDF's acquisition by PDFTron (Apryse) in June 2020, which raised understandable questions about long-term roadmap and licensing continuity. If you're in that position, this guide walks you through the mechanical steps of replacing Toolkit API calls with IronPDF equivalents, with working before-and-after code for the most common operations.

Why Teams Leave ActivePDF

ActivePDF Toolkit is a capable library, but several friction points accumulate over time. Its stateful Open/Close pattern — toolkit.OpenOutputFile() followed eventually by toolkit.CloseOutputFile() — doesn't compose well with modern async .NET code. Error handling is integer return-code based rather than exception based, which makes it awkward to fit into standard try/catch flows. Installation historically required manual DLL references and filesystem path configuration rather than a simple NuGet restore. And machine-locked licensing complicates cloud and containerised deployments.

IronPDF addresses each of these: it's a NuGet package, uses exceptions, is stateless (no open/close), and uses a code-based license key that works in any environment.

Installation and Setup

Remove the old package and add the new one:

# Remove ActivePDF
dotnet remove package APToolkitNET

# Add IronPDF
dotnet add package IronPdf

Set your license key at application startup, before any PDF operations:

IronPdf.License.LicenseKey = "YOUR-LICENSE-KEY";

Namespace and Using Statement Mapping

  • using ActivePDF.Toolkit; → using IronPdf; — core namespace
  • using APToolkitNET; → using IronPdf;
  • using APToolkitNET.PDFObjects; → using IronPdf;
  • using APToolkitNET.Common; → using IronPdf;
  • using APToolkitNET.Extractor; → using IronPdf; — text extraction
  • using APToolkitNET.Rasterizer; → using IronPdf; — PDF to image

The global find-and-replace on using statements handles most of the namespace surface; the remaining work is converting the API calls themselves.

Core API Mapping

  • Document creation. new Toolkit() → new ChromePdfRenderer(). No path argument needed.
  • Open/close pattern. toolkit.OpenOutputFile(path) / toolkit.CloseOutputFile() → no equivalent; just call pdf.SaveAs(path) at the end.
  • HTML to PDF. toolkit.AddHTML(html) → renderer.RenderHtmlAsPdf(html), returns a PdfDocument.
  • URL to PDF. toolkit.AddURL(url) → renderer.RenderUrlAsPdf(url).
  • Load existing PDF. toolkit.OpenInputFile(path) → PdfDocument.FromFile(path).
  • Merge. toolkit.AddPDF(path) in a loop → PdfDocument.Merge(pdfs) with a list of loaded documents.
  • Page count. toolkit.GetPageCount() → pdf.PageCount.
  • Delete page. toolkit.DeletePage(index) → pdf.RemovePages(index).
  • Text extraction. toolkit.GetText() → pdf.ExtractAllText(). Per-page: pdf.Pages[i].Text.
  • Watermark. toolkit.AddWatermark(text, angle, opacity) → pdf.ApplyWatermark(html, rotation, opacity) where opacity is an integer 0–100.
  • Encryption. toolkit.Encrypt(pwd) / toolkit.SetUserPassword(pwd) → pdf.SecuritySettings.OwnerPassword / pdf.SecuritySettings.UserPassword.
  • Page size. ActivePDF takes points (612, 792 = Letter). IronPDF uses RenderingOptions.PaperSize = PdfPaperSize.Letter or SetCustomPaperSizeInMillimeters().
  • Error handling. ActivePDF returns integer codes; IronPDF throws exceptions. Replace if (result != 0) checks with try/catch blocks.

HTML String to PDF

The most common operation: render an HTML string directly to a PDF file.

Before (ActivePDF Toolkit):

// NuGet: Install-Package APToolkitNET
using ActivePDF.Toolkit;

class Program
{
    static void Main()
    {
        Toolkit toolkit = new Toolkit();

        string htmlContent = "<html><body><h1>Hello World</h1></body></html>";

        if (toolkit.OpenOutputFile("output.pdf") == 0)
        {
            toolkit.AddHTML(htmlContent);
            toolkit.CloseOutputFile();
            Console.WriteLine("PDF created successfully");
        }
    }
}

After (IronPDF):

// NuGet: Install-Package IronPdf
using IronPdf;

class Program
{
    static void Main()
    {
        var renderer = new ChromePdfRenderer();

        string htmlContent = "<html><body><h1>Hello World</h1></body></html>";

        var pdf = renderer.RenderHtmlAsPdf(htmlContent);
        pdf.SaveAs("output.pdf");

        Console.WriteLine("PDF created successfully");
    }
}

The open/close calls disappear entirely. RenderHtmlAsPdf returns a fully-formed PdfDocument; SaveAs writes it to disk.

URL to PDF

Fetching a live web page and converting it to PDF.

Before (ActivePDF Toolkit):

// NuGet: Install-Package APToolkitNET
using ActivePDF.Toolkit;

class Program
{
    static void Main()
    {
        Toolkit toolkit = new Toolkit();

        string url = "https://www.example.com";

        if (toolkit.OpenOutputFile("webpage.pdf") == 0)
        {
            toolkit.AddURL(url);
            toolkit.CloseOutputFile();
            Console.WriteLine("PDF from URL created successfully");
        }
    }
}

After (IronPDF):

// NuGet: Install-Package IronPdf
using IronPdf;

class Program
{
    static void Main()
    {
        var renderer = new ChromePdfRenderer();

        string url = "https://www.example.com";

        var pdf = renderer.RenderUrlAsPdf(url);
        pdf.SaveAs("webpage.pdf");

        Console.WriteLine("PDF from URL created successfully");
    }
}

IronPDF uses a headless Chromium engine under the hood, so JavaScript-heavy pages render correctly. If a page depends on client-side rendering, add a wait:

renderer.RenderingOptions.WaitFor.RenderDelay(2000);

Merging Multiple PDFs

ActivePDF merges by calling AddPDF once per file inside an open output file context. IronPDF loads each document first, then merges the collection.

Before (ActivePDF Toolkit):

// NuGet: Install-Package APToolkitNET
using ActivePDF.Toolkit;

class Program
{
    static void Main()
    {
        Toolkit toolkit = new Toolkit();

        if (toolkit.OpenOutputFile("merged.pdf") == 0)
        {
            toolkit.AddPDF("document1.pdf");
            toolkit.AddPDF("document2.pdf");
            toolkit.CloseOutputFile();
            Console.WriteLine("PDFs merged successfully");
        }
    }
}

After (IronPDF):

// NuGet: Install-Package IronPdf
using IronPdf;
using System.Collections.Generic;

class Program
{
    static void Main()
    {
        var pdf1 = PdfDocument.FromFile("document1.pdf");
        var pdf2 = PdfDocument.FromFile("document2.pdf");

        var merged = PdfDocument.Merge(pdf1, pdf2);
        merged.SaveAs("merged.pdf");

        Console.WriteLine("PDFs merged successfully");
    }
}

For a variable-length list, use the overload that takes an IEnumerable<PdfDocument>:

using System.Linq;

var pdfs = inputFiles.Select(PdfDocument.FromFile).ToList();
using var merged = PdfDocument.Merge(pdfs);
merged.SaveAs("merged.pdf");
foreach (var pdf in pdfs) pdf.Dispose();

Password Protection

Before (ActivePDF Toolkit):

using ActivePDF.Toolkit;

public void ProtectPdf(string inputPath, string outputPath, string password)
{
    Toolkit toolkit = new Toolkit();

    if (toolkit.OpenInputFile(inputPath) == 0)
    {
        toolkit.Encrypt(password);
        toolkit.SetUserPassword(password);
        toolkit.SetPermissions(4); // Print only
        toolkit.SaveAs(outputPath);
        toolkit.CloseInputFile();
    }
}

After (IronPDF):

using IronPdf;

public void ProtectPdf(string inputPath, string outputPath, string password)
{
    using var pdf = PdfDocument.FromFile(inputPath);

    pdf.SecuritySettings.OwnerPassword = password;
    pdf.SecuritySettings.UserPassword = password;
    pdf.SecuritySettings.AllowUserPrinting = PdfPrintSecurity.FullPrintRights;
    pdf.SecuritySettings.AllowUserCopyPasteContent = false;
    pdf.SecuritySettings.AllowUserEdits = PdfEditSecurity.NoEdit;

    pdf.SaveAs(outputPath);
}

Watermarking

Before (ActivePDF Toolkit):

using ActivePDF.Toolkit;

public void AddWatermark(string inputPath, string outputPath, string watermarkText)
{
    Toolkit toolkit = new Toolkit();

    if (toolkit.OpenInputFile(inputPath) == 0)
    {
        int pageCount = toolkit.GetPageCount();
        for (int i = 1; i <= pageCount; i++)
        {
            toolkit.SetPage(i);
            toolkit.AddWatermark(watermarkText, 45, 0.5f);
        }
        toolkit.SaveAs(outputPath);
        toolkit.CloseInputFile();
    }
}

After (IronPDF):

using IronPdf;

public void AddWatermark(string inputPath, string outputPath, string watermarkText)
{
    using var pdf = PdfDocument.FromFile(inputPath);

    pdf.ApplyWatermark(
        $"<h1 style='color:lightgray;font-size:72px;'>{watermarkText}</h1>",
        rotation: 45,
        opacity: 50);

    pdf.SaveAs(outputPath);
}

Note that IronPDF's opacity parameter is an integer from 0 to 100, not a float.

Text Extraction

Before (ActivePDF Toolkit):

using ActivePDF.Toolkit;

public string ExtractText(string pdfPath)
{
    Toolkit toolkit = new Toolkit();
    string text = "";

    if (toolkit.OpenInputFile(pdfPath) == 0)
    {
        int pageCount = toolkit.GetPageCount();
        for (int i = 1; i <= pageCount; i++)
        {
            text += toolkit.GetTextFromPage(i) + "\n";
        }
        toolkit.CloseInputFile();
    }

    return text;
}

After (IronPDF):

using IronPdf;

public string ExtractText(string pdfPath)
{
    using var pdf = PdfDocument.FromFile(pdfPath);
    return pdf.ExtractAllText();
}

ASP.NET Core Integration

Before (ActivePDF Toolkit):

[HttpPost]
public IActionResult GeneratePdf([FromBody] ReportRequest request)
{
    Toolkit toolkit = new Toolkit();

    if (toolkit.OpenOutputFile("temp.pdf") == 0)
    {
        toolkit.AddHTML(request.Html);
        toolkit.CloseOutputFile();

        byte[] bytes = System.IO.File.ReadAllBytes("temp.pdf");
        return File(bytes, "application/pdf", "report.pdf");
    }

    return BadRequest("PDF generation failed");
}

After (IronPDF):

[HttpPost]
public IActionResult GeneratePdf([FromBody] ReportRequest request)
{
    var renderer = new ChromePdfRenderer();
    using var pdf = renderer.RenderHtmlAsPdf(request.Html);

    return File(pdf.BinaryData, "application/pdf", "report.pdf");
}

For async controllers:

public async Task<IActionResult> GeneratePdf([FromBody] ReportRequest request)
{
    var renderer = new ChromePdfRenderer();
    using var pdf = await renderer.RenderHtmlAsPdfAsync(request.Html);

    return File(pdf.BinaryData, "application/pdf", "report.pdf");
}

Key Differences to Watch For

  • Page indexing. ActivePDF Toolkit is 1-based throughout (GetTextFromPage(1) is page one). IronPDF's pdf.Pages collection is 0-based.
  • Page size units. ActivePDF takes points (612 × 792 = US Letter). IronPDF uses enums (PdfPaperSize.Letter) or millimetres via SetCustomPaperSizeInMillimeters().
  • Error handling. Replace integer return-code checks with try/catch; IronPDF throws exceptions on failure.
  • Opacity type. ApplyWatermark takes an integer (0–100), not a float.
  • Renderer reuse. ChromePdfRenderer is safe to reuse across calls and should be treated as a long-lived object in server applications to avoid repeated Chrome engine startup cost.
  • Licensing. Set IronPdf.License.LicenseKey at application startup before any PDF operation. Machine-locked licensing is not required.

Common Migration Pitfalls

  • Stale DLL references. After removing the NuGet package, check your .csproj for any remaining <Reference Include="APToolkitNET"> hint-path entries and delete them.
  • Off-by-one in page loops. If you iterated for (int i = 1; i <= pageCount; i++) in ActivePDF, switch to for (int i = 0; i < pdf.PageCount; i++) in IronPDF.
  • Blank pages from JS-heavy URLs. Enable a render delay or element-wait if content doesn't appear: renderer.RenderingOptions.WaitFor.HtmlElementById("content-loaded").
  • First-call latency. The Chromium engine initialises on the first render call (5–10 seconds). Warm it up at startup with a no-op render if cold-start latency matters.

Migration Checklist

  • Audit all using ActivePDF.Toolkit; and using APToolkitNET references
  • Remove APToolkitNET NuGet package and any manual DLL hint-path references
  • Install IronPdf NuGet package
  • Set IronPdf.License.LicenseKey at application startup
  • Replace new Toolkit() with new ChromePdfRenderer()
  • Remove all OpenOutputFile / CloseOutputFile / OpenInputFile / CloseInputFile calls
  • Replace integer return-code checks with try/catch blocks
  • Convert page size from points to enums or millimetres
  • Adjust page-index loops from 1-based to 0-based
  • Convert watermark opacity from float to integer
  • Run and compare PDF outputs visually in staging

The mechanical conversion is straightforward once you see the pattern: ActivePDF's stateful open-operate-close cycle collapses into IronPDF's functional render-then-save model. The three operations most teams rely on — HTML rendering, URL capture, and PDF merging — map directly with no missing functionality.
