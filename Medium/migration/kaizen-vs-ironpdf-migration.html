I Replaced Our Kaizen.io HTML-to-PDF Integration with IronPDF — Here's the Honest Breakdown

By Jacob Mellor · IronPDF Engineering

A client's staging environment had no outbound internet access. Their PDF generation — wired to the Kaizen.IO NuGet SDK — stopped dead. The converter threw a connection exception before it could produce a single byte. That was the moment the team decided to move to something that runs entirely in-process.

This article covers what that migration actually looks like: the API differences, the code changes, and the things worth knowing before you start.

Why Teams Move Away from Kaizen.io

Kaizen.IO HTML-to-PDF is a .NET SDK that wraps an external service call. It works well in connected environments. But teams hit friction when those assumptions break:

  • Network dependency. Every PDF generation requires a successful outbound call. Airgapped environments, VPNs with egress restrictions, and staging environments without internet access all become blockers.
  • Data leaves your infrastructure. HTML content — including any sensitive data embedded in it — is transmitted externally. Healthcare, finance, and legal teams often cannot accept that.
  • Per-request cost model. Volume spikes translate directly to invoice spikes. A $50 one-time license sounds appealing until your batch job generates 50,000 PDFs in a night.
  • API key management. Every call must carry an API key. Rotation, revocation, and secret management add operational surface area.
  • Rate limits and retry logic. Cloud services throttle. Teams end up maintaining exponential backoff, retry queues, and 429 handlers that have nothing to do with PDF generation.

IronPDF runs in-process via Chromium. No outbound call, no key per request, no rate limits. The trade-off is a heavier NuGet dependency and a Chromium warm-up cost on first render.

Installation

# Remove Kaizen.io
dotnet remove package Kaizen.IO.HtmlToPdf

# Install IronPDF
dotnet add package IronPdf

Set the IronPDF license key once at application startup — Program.cs or Startup.cs:

IronPdf.License.LicenseKey = "YOUR-LICENSE-KEY";

Without a key, IronPDF renders with a watermark (free evaluation mode). Kaizen.io used an API key passed per-request to the constructor; that entire pattern is gone.

Basic HTML to PDF

The minimal migration. Kaizen.IO takes an HTML string and returns byte[]. IronPDF returns a PdfDocument; call .BinaryData or .SaveAs() to get bytes or write to disk.

Before (Kaizen.IO):

using Kaizen.IO;
using System.IO;

class Program
{
    static void Main()
    {
        var converter = new HtmlToPdfConverter();
        var html = "<html><body><h1>Hello World</h1></body></html>";
        var pdfBytes = converter.Convert(html);
        File.WriteAllBytes("output.pdf", pdfBytes);
    }
}

After (IronPDF):

// NuGet: Install-Package IronPdf
using IronPdf;
using System.IO;

class Program
{
    static void Main()
    {
        var renderer = new ChromePdfRenderer();
        var html = "<html><body><h1>Hello World</h1></body></html>";
        var pdf = renderer.RenderHtmlAsPdf(html);
        pdf.SaveAs("output.pdf");
    }
}

The structural change: HtmlToPdfConverter → ChromePdfRenderer, converter.Convert(html) → renderer.RenderHtmlAsPdf(html). No API key argument to the constructor.

HTML File to PDF with Page Options

When converting an HTML file from disk, Kaizen.IO accepts a ConversionOptions object. IronPDF exposes equivalent settings on renderer.RenderingOptions before calling RenderHtmlFileAsPdf().

Before (Kaizen.IO):

using Kaizen.IO;
using System;
using System.IO;

class Program
{
    static void Main()
    {
        var converter = new HtmlToPdfConverter();
        var htmlContent = File.ReadAllText("input.html");
        var options = new ConversionOptions
        {
            PageSize = PageSize.A4,
            Orientation = Orientation.Portrait
        };
        var pdfBytes = converter.Convert(htmlContent, options);
        File.WriteAllBytes("document.pdf", pdfBytes);
    }
}

After (IronPDF):

// NuGet: Install-Package IronPdf
using IronPdf;
using System;
using System.IO;

class Program
{
    static void Main()
    {
        var renderer = new ChromePdfRenderer();
        renderer.RenderingOptions.PaperSize = PdfPaperSize.A4;
        renderer.RenderingOptions.PaperOrientation = PdfPaperOrientation.Portrait;
        var pdf = renderer.RenderHtmlFileAsPdf("input.html");
        pdf.SaveAs("document.pdf");
    }
}

Use RenderHtmlFileAsPdf(path) rather than reading the file into a string first — it lets IronPDF resolve relative asset paths (images, CSS) correctly using the file's directory as the base URL.

URL to PDF with Headers and Footers

Kaizen.IO supports HTML-based header and footer content via HeaderOptions and FooterOptions. IronPDF's equivalent is TextHeader and TextFooter on RenderingOptions for simple text, or HtmlHeader / HtmlFooter for full HTML fragments. The placeholder for total pages differs: Kaizen uses {total}; IronPDF uses {total-pages}.

Before (Kaizen.IO):

using Kaizen.IO;
using System;
using System.IO;

class Program
{
    static void Main()
    {
        var converter = new HtmlToPdfConverter();
        var options = new ConversionOptions
        {
            Header = new HeaderOptions { HtmlContent = "<div style='text-align:center'>Company Header</div>" },
            Footer = new FooterOptions { HtmlContent = "<div style='text-align:center'>Page {page} of {total}</div>" },
            MarginTop = 20,
            MarginBottom = 20
        };
        var pdfBytes = converter.ConvertUrl("https://example.com", options);
        File.WriteAllBytes("webpage.pdf", pdfBytes);
    }
}

After (IronPDF):

// NuGet: Install-Package IronPdf
using IronPdf;
using System;
using System.IO;

class Program
{
    static void Main()
    {
        var renderer = new ChromePdfRenderer();
        renderer.RenderingOptions.TextHeader.CenterText = "Company Header";
        renderer.RenderingOptions.TextFooter.CenterText = "Page {page} of {total-pages}";
        renderer.RenderingOptions.MarginTop = 20;
        renderer.RenderingOptions.MarginBottom = 20;
        var pdf = renderer.RenderUrlAsPdf("https://example.com");
        pdf.SaveAs("webpage.pdf");
    }
}

Note the placeholder change: {total} becomes {total-pages}. Also note that TextHeader.CenterText is plain text; if you need full HTML in headers, switch to HtmlHeader = new HtmlHeaderFooter { HtmlFragment = "..." }.

API Mapping Reference

Class and Constructor Mappings

  • HtmlToPdfConverter() — no constructor arg needed → new ChromePdfRenderer()
  • HtmlToPdfConverter(apiKey) → set IronPdf.License.LicenseKey once at startup instead
  • ConversionOptions → configure renderer.RenderingOptions properties directly
  • HeaderOptions { HtmlContent = ... } → renderer.RenderingOptions.HtmlHeader = new HtmlHeaderFooter { HtmlFragment = ... }
  • FooterOptions { HtmlContent = ... } → renderer.RenderingOptions.HtmlFooter = new HtmlHeaderFooter { HtmlFragment = ... }

Method Mappings

  • converter.Convert(html) → renderer.RenderHtmlAsPdf(html)
  • converter.Convert(htmlString, options) → set RenderingOptions then renderer.RenderHtmlAsPdf(html)
  • converter.ConvertUrl(url, options) → set RenderingOptions then renderer.RenderUrlAsPdf(url)
  • converter.ConvertFile(path) → renderer.RenderHtmlFileAsPdf(path)
  • converter.ConvertAsync(...) → renderer.RenderHtmlAsPdfAsync(...)
  • Return value is PdfDocument, not byte[] — use pdf.BinaryData or pdf.SaveAs(path)

ConversionOptions Property Mappings

  • PageSize = PageSize.A4 → RenderingOptions.PaperSize = PdfPaperSize.A4
  • Orientation = Orientation.Portrait → RenderingOptions.PaperOrientation = PdfPaperOrientation.Portrait
  • MarginTop / MarginBottom / MarginLeft / MarginRight → same names on RenderingOptions, in millimeters
  • Header.HtmlContent → RenderingOptions.HtmlHeader.HtmlFragment
  • Footer.HtmlContent → RenderingOptions.HtmlFooter.HtmlFragment
  • BaseUrl → RenderingOptions.BaseUrl
  • EnableJavaScript → RenderingOptions.EnableJavaScript (default true)
  • JavaScriptDelay = 3000 → RenderingOptions.RenderDelay = 3000
  • PrintBackground → RenderingOptions.PrintHtmlBackgrounds
  • Scale → RenderingOptions.Zoom
  • Timeout → RenderingOptions.Timeout

Placeholder Mappings

  • {page} → {page} — unchanged
  • {total} → {total-pages}
  • {date} → {date}
  • {time} → {time}
  • {title} → {html-title}
  • {url} → {url}

Async Operations

IronPDF mirrors Kaizen.IO's async pattern. The call site change is minimal:

// Before (Kaizen.IO)
var pdfBytes = await converter.ConvertAsync(html, options);

// After (IronPDF)
var pdf = await renderer.RenderHtmlAsPdfAsync(html);
byte[] pdfBytes = pdf.BinaryData;

Error Handling Changes

Kaizen.IO code typically catches ApiException, HttpRequestException, and 429 rate-limit responses. All of that goes away with IronPDF — there is no outbound network call to fail.

// Before (Kaizen.IO)
try
{
    return await converter.ConvertAsync(html, options);
}
catch (ApiException ex) when (ex.StatusCode == 429)
{
    await Task.Delay(1000);
    return await converter.ConvertAsync(html, options);  // Retry
}
catch (HttpRequestException ex)
{
    throw new Exception("Network error: " + ex.Message);
}

// After (IronPDF)
try
{
    var pdf = await renderer.RenderHtmlAsPdfAsync(html);
    return pdf.BinaryData;
}
catch (IronPdf.Exceptions.IronPdfLicensingException ex)
{
    throw new Exception("License error: " + ex.Message);
}
// No network errors, no rate limits, no 429 handling needed.

Delete all retry logic, backoff timers, and API-availability checks. The only IronPDF-specific exception class worth catching at startup is IronPdfLicensingException if the key is invalid.

Performance Considerations

The one genuine caveat: IronPDF spins up a Chromium instance on first use. That first render is slower than Kaizen.IO's (which returns a cloud-rendered result). Subsequent renders are fast. Warm up at startup:

// In Program.cs or Startup.cs — before serving requests
new ChromePdfRenderer().RenderHtmlAsPdf("<html></html>");

After warm-up, local Chromium rendering is competitive with a cloud round-trip for complex pages — and faster for simple ones because there is no network latency.

Reuse the ChromePdfRenderer instance across requests rather than constructing it per call. Parallel rendering works without throttling: IronPDF has no rate limits, so Task.WhenAll() across concurrent renders is safe.

Licensing

Kaizen.io sold a $50 one-time license; the free tier watermarks output. IronPDF uses a similar model — free evaluation with a watermark, one-time or annual commercial license for watermark-free production use. The key is set in code rather than passed to a Docker container via environment variable.

// Set once — before any rendering call
IronPdf.License.LicenseKey = "YOUR-LICENSE-KEY";

Migration Checklist

Pre-Migration

  • Find all using Kaizen.IO references across the codebase — those files need updates.
  • List every ConversionOptions configuration in use — margins, page size, orientation, headers, footers, JS delay — so you can verify equivalents.
  • Note header/footer template strings. Update {total} → {total-pages} and {title} → {html-title}.
  • Obtain an IronPDF license key for production.

Package Changes

  • Remove Kaizen.IO.HtmlToPdf (or equivalent package name) from every project that references it.
  • Add IronPdf NuGet package.
  • Replace using Kaizen.IO; with using IronPdf;.

Code Changes

  • Add IronPdf.License.LicenseKey = "YOUR-LICENSE-KEY"; at application startup.
  • Replace new HtmlToPdfConverter() with new ChromePdfRenderer().
  • Move ConversionOptions properties to renderer.RenderingOptions.*.
  • Replace converter.Convert(html) with renderer.RenderHtmlAsPdf(html).
  • Replace converter.ConvertUrl(url) with renderer.RenderUrlAsPdf(url).
  • Replace converter.ConvertFile(path) with renderer.RenderHtmlFileAsPdf(path).
  • Replace File.WriteAllBytes(path, pdfBytes) with pdf.SaveAs(path), or read pdf.BinaryData where a byte array is required downstream.
  • Update placeholder strings: {total} → {total-pages}, {title} → {html-title}.
  • Delete all 429 retry logic, HttpRequestException handlers, and rate-limit tracking.
  • Remove API key from configuration, secrets manager, and DI setup.

Testing

  • Verify all HTML-to-PDF paths produce correct output.
  • Confirm header and footer text renders with correct page numbers.
  • Validate margins and page size match the original output.
  • Test async paths under concurrent load — no throttling expected.
  • Run in an environment without internet access to confirm offline operation.

Post-Migration

  • Remove the Kaizen.io API key from environment variables and secrets stores.
  • Remove any monitoring or alerting tied to Kaizen.io API error rates or latency.
  • Update runbook error-handling documentation to remove 429 and network-error scenarios.
