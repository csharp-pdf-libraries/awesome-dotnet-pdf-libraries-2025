Docotic.Pdf vs IronPDF in C#: Which PDF Library Should Your Team Choose?

Two architectural choices shape day-one experience with BitMiracle Docotic.Pdf. First, HTML-to-PDF is not part of the core package: it lives in a separate BitMiracle.Docotic.Pdf.HtmlToPdf add-on built on Chromium with an async-only API. Second, licensing is applied via LicenseManager.AddLicenseData(...) before production use, with a 31-day trial obtained by request. Compared with IronPDF's single-package install and "watermark until licensed" trial model, that is a different developer-onboarding shape.

These are the kinds of architectural differences that compound over a project: package surface, async model, how HTML-to-PDF is exposed, and how each library wants to be deployed.

I want to walk through what each library offers, how they differ, and where one fits better than the other so your team can make a confident decision.

What Is Docotic.Pdf?

BitMiracle Docotic.Pdf is an actively maintained .NET PDF library with consistent releases and recent .NET support. Its product focus is broad PDF manipulation — text extraction, form handling, and page editing — shipped as the BitMiracle.Docotic.Pdf NuGet package. HTML-to-PDF rendering is provided by a separate Chromium-based add-on, BitMiracle.Docotic.Pdf.HtmlToPdf, exposed through HtmlConverter.CreateAsync() and async-only methods like CreatePdfFromStringAsync(html).

The core library is 100% managed code with no native dependencies, which is an architectural strength on Linux and constrained hosting. The HtmlToPdf add-on, by contrast, is Chromium-based and downloads a browser binary on first use, so any HTML rendering scenario brings native binaries with it.

Licensing is applied via LicenseManager.AddLicenseData(key). Trial licenses are issued for 31 days on request, and production use requires a paid license — a different posture than IronPDF's "install and start, watermark until licensed" trial.

How Does Docotic.Pdf Differ from IronPDF?

The core difference is package surface. Docotic.Pdf is a managed-code PDF manipulation library; HTML-to-PDF rendering lives in a separate Chromium-based BitMiracle.Docotic.Pdf.HtmlToPdf add-on with its own async-only API surface. IronPDF standardizes on Chromium for all HTML rendering in a single package, so HTML-to-PDF, URL-to-PDF, and async rendering live behind one API.

The ChromePdfRenderer class in IronPDF works without configuration for development and testing, applying a watermark until licensed. Docotic.Pdf expects a license key (typically obtained as a trial via email request) for production use.

Installation favors IronPDF for time-to-first-PDF: one NuGet package and you can start coding immediately. Docotic.Pdf's manipulation flow is single-package (BitMiracle.Docotic.Pdf); HTML rendering pulls in the BitMiracle.Docotic.Pdf.HtmlToPdf add-on as a second package.

Async support is another differentiator. IronPDF provides native async methods like RenderHtmlAsPdfAsync alongside synchronous equivalents on the same renderer. Docotic.Pdf's HtmlToPdf add-on is async-only — HtmlConverter.CreateAsync(), CreatePdfFromStringAsync(html), CreatePdfAsync(uri) — and the core PDF manipulation API is synchronous.

How Do You Convert HTML to PDF with Each Library?

Docotic.Pdf — HTML to PDF (HtmlToPdf add-on)

// NuGet: Install-Package BitMiracle.Docotic.Pdf
// NuGet: Install-Package BitMiracle.Docotic.Pdf.HtmlToPdf  (separate add-on required)
using System;
using System.Threading.Tasks;
using BitMiracle.Docotic.Pdf;

class Program
{
    static async Task Main()
    {
        // HtmlConverter is async-only and downloads Chromium on first use
        using var converter = await HtmlConverter.CreateAsync();

        string html = @"
            <html>
            <head>
                <style>
                    body { font-family: Arial; margin: 20px; }
                    h1 { color: #333; }
                </style>
            </head>
            <body>
                <h1>Hello from Docotic.Pdf</h1>
                <p>Sample content with <strong>formatting</strong>.</p>
            </body>
            </html>";

        using var pdf = await converter.CreatePdfFromStringAsync(html);
        pdf.Save("output.pdf");

        Console.WriteLine("PDF created successfully");
    }
}

The HtmlToPdf add-on uses Chromium under the hood, so modern CSS, web fonts, and JavaScript-driven layouts are supported through the same engine class.

IronPDF — Basic HTML to PDF

using System;
using IronPdf;

namespace IronPdfHtmlExample
{
    class Program
    {
        static void Main(string[] args)
        {
            IronPdf.License.LicenseKey = "YOUR-LICENSE-KEY";

            // Create renderer
            var renderer = new ChromePdfRenderer();

            // Render HTML
            string html = @"
                <html>
                <head>
                    <style>
                        body { font-family: Arial; margin: 20px; }
                        h1 { color: #333; }
                    </style>
                </head>
                <body>
                    <h1>Hello from IronPDF</h1>
                    <p>Sample content with <strong>formatting</strong>.</p>
                </body>
                </html>";

            var pdf = renderer.RenderHtmlAsPdf(html);

            // Save
            pdf.SaveAs("output.pdf");
        }
    }
}

Three steps: create renderer, render HTML, save. For more HTML string details, see the HTML string to PDF guide.

How Do Merging, Signing, Forms, and Advanced Features Compare?

Docotic.Pdf is strong at PDF manipulation. Text extraction, form field handling, and PDF editing are all in its core. If your workflow centers on manipulating existing PDFs, Docotic.Pdf's feature set is robust. Merging is a real native operation on PdfDocument — pdf1.Append("file.pdf") accepts a file path, Stream, or byte[], then Save.

The trade-off shows up in HTML-focused workflows. URL-to-PDF, async rendering, and header/footer HTML templating are first-class in IronPDF on a single renderer, while in Docotic.Pdf they live behind the separate HtmlToPdf add-on with its own async-only API surface.

Here is URL-to-PDF in each library.

Docotic.Pdf — URL to PDF (HtmlToPdf add-on)

// NuGet: Install-Package BitMiracle.Docotic.Pdf
// NuGet: Install-Package BitMiracle.Docotic.Pdf.HtmlToPdf
using System;
using System.Threading.Tasks;
using BitMiracle.Docotic.Pdf;

class Program
{
    static async Task Main()
    {
        using var converter = await HtmlConverter.CreateAsync();

        var options = new HtmlConversionOptions();
        options.Page.SetSize(PdfPaperSize.A4, isLandscape: true);

        using var pdf = await converter.CreatePdfAsync(
            new Uri("https://example.com"), options);

        pdf.Save("url_output.pdf");
        Console.WriteLine("URL converted");
    }
}

IronPDF — URL to PDF

using System;
using IronPdf;

namespace IronPdfUrlExample
{
    class Program
    {
        static void Main(string[] args)
        {
            IronPdf.License.LicenseKey = "YOUR-LICENSE-KEY";

            var renderer = new ChromePdfRenderer();

            renderer.RenderingOptions.Timeout = 120;

            var pdf = renderer.RenderUrlAsPdf("https://example.com");

            pdf.SaveAs("url_output.pdf");
        }
    }
}

For URL rendering with authentication, see the URL to PDF documentation.

Async Operations

Async support matters for web applications. IronPDF provides native async methods:

using System;
using System.Threading.Tasks;
using IronPdf;

namespace IronPdfAsyncExample
{
    class Program
    {
        static async Task Main(string[] args)
        {
            IronPdf.License.LicenseKey = "YOUR-LICENSE-KEY";

            var renderer = new ChromePdfRenderer();

            var pdf = await renderer.RenderHtmlAsPdfAsync("<h1>Async</h1>");

            await pdf.SaveAsAsync("async_output.pdf");
        }
    }
}

With Docotic.Pdf, the core PDF manipulation API is synchronous; the HtmlToPdf add-on is async-only, exposing methods like HtmlConverter.CreateAsync(), CreatePdfFromStringAsync(html), and CreatePdfAsync(uri).

How Do You Install Each Library?

Docotic.Pdf Installation

# Step 1: Install core
dotnet add package BitMiracle.Docotic.Pdf

# Step 2 (only if you need HTML rendering):
# dotnet add package BitMiracle.Docotic.Pdf.HtmlToPdf

# Step 3: Request trial license (or apply your purchased key)
# Visit: https://bitmiracle.com/pdf-library/

using BitMiracle.Docotic.Pdf;

// Apply license before production use
LicenseManager.AddLicenseData("YOUR-LICENSE-KEY");

dotnet add package IronPdf

How Do the Features Compare Side by Side?

  • Product focus. Docotic.Pdf leans into broad PDF manipulation (text extraction, forms, page editing); IronPDF leans into HTML-to-PDF rendering quality. Both are actively developed.
  • HTML rendering engine. IronPDF is Chromium-based for all HTML rendering, included in the main package. Docotic.Pdf provides Chromium-based HTML rendering through the separate BitMiracle.Docotic.Pdf.HtmlToPdf add-on.
  • Package surface. IronPDF ships as a single IronPdf NuGet package. Docotic.Pdf's manipulation flow is single-package (BitMiracle.Docotic.Pdf); HTML rendering pulls in BitMiracle.Docotic.Pdf.HtmlToPdf.
  • HTML / CSS / JS. Both engines are Chromium-based once the Docotic add-on is installed, so HTML5, CSS3, and JavaScript support is comparable in scope. The difference is whether that engine is in the core package or in a second package.
  • URL-to-PDF and async. First-class in IronPDF (RenderUrlAsPdf, RenderHtmlAsPdfAsync) on a single renderer. In Docotic.Pdf, URL rendering and async sit in the HtmlToPdf add-on as CreatePdfAsync(uri) on HtmlConverter.
  • PDF manipulation. Docotic.Pdf has a comprehensive surface for text extraction, form-field editing, and page-level manipulation. IronPDF supports merge, copy, watermark, sign, and form filling — see the IronPDF API reference.
  • Native dependencies. The Docotic.Pdf core is 100% managed code with no native binaries. The HtmlToPdf add-on, like IronPDF, ships a Chromium engine — once HTML rendering is in scope, both libraries bring native binaries to manage on Linux and constrained hosts.
  • Licensing posture. Docotic.Pdf expects a license key applied via LicenseManager.AddLicenseData and a 31-day trial obtained on request. IronPDF runs without a key (watermarked) until you apply IronPdf.License.LicenseKey.

Frequently Asked Questions

Can I use Docotic.Pdf without a license key?

For production use, Docotic.Pdf expects a license key applied via LicenseManager.AddLicenseData(key). A 31-day trial key is issued on request. IronPDF, by contrast, runs immediately without a key and watermarks output until IronPdf.License.LicenseKey is set.

Do both libraries use Chromium for HTML rendering?

Yes — once HTML rendering is in scope, both are Chromium-based. IronPDF includes Chromium in its main package. Docotic.Pdf delivers Chromium through the separate BitMiracle.Docotic.Pdf.HtmlToPdf add-on, accessed via HtmlConverter.CreateAsync().

Which library is better for PDF manipulation tasks like text extraction and form filling?

Docotic.Pdf has a strong surface for text extraction and form-field handling — that is squarely in its core. Teams whose primary workflow is editing existing PDFs, extracting text, and filling forms tend to find Docotic.Pdf's feature set well-suited to those tasks.

Does Docotic.Pdf support async operations for HTML-to-PDF conversion?

The HtmlToPdf add-on is async-only — HtmlConverter.CreateAsync(), CreatePdfFromStringAsync(html), and CreatePdfAsync(uri) are the entry points. The core PDF manipulation API is synchronous. IronPDF provides native async methods such as RenderHtmlAsPdfAsync and SaveAsAsync alongside synchronous equivalents on its single rendering surface.

How many NuGet packages does each library require for HTML-to-PDF?

Docotic.Pdf requires two packages for HTML-to-PDF: BitMiracle.Docotic.Pdf for core PDF operations and BitMiracle.Docotic.Pdf.HtmlToPdf for the Chromium-based HTML converter. IronPDF requires one package (IronPdf) that covers HTML-to-PDF, URL-to-PDF, and async rendering in one API.

What happens if I forget to set the Docotic.Pdf license key?

Without a valid key applied via LicenseManager.AddLicenseData(key), Docotic.Pdf surfaces a license validation error rather than silently watermarking. The exact error text and edge-case behavior can vary across versions, so check the current documentation for specifics.

Which Library Should Your Team Choose?

BitMiracle Docotic.Pdf is a comprehensive .NET PDF manipulation library with strong text extraction, form-field handling, and editing capabilities, plus an HtmlToPdf add-on that uses Chromium for HTML rendering. Teams whose primary workflow is manipulating existing PDFs — and whose core dependencies stay 100% managed code unless HTML rendering is needed — will find Docotic.Pdf's architecture a natural fit.

The trade-offs to weigh are: a two-package surface for HTML rendering (BitMiracle.Docotic.Pdf plus the BitMiracle.Docotic.Pdf.HtmlToPdf add-on), a license-first posture for production, and an async-only HTML converter while the core API is synchronous. None of these are blockers; they are decisions your team should make explicitly rather than discover mid-project.

If your codebase tilts toward HTML-to-PDF, URL-to-PDF, and async rendering more than toward PDF manipulation, IronPDF's single-package, single-rendering-surface model removes some of that surface area. The ChromePdfRenderer works immediately for development without license configuration, watermarking output until licensed.
